1. What is Flexbox?

Flexbox (Flexible Box Layout) is a CSS layout method that makes it easy to arrange items in a row or column, distribute space between them, and align them — even when their sizes are unknown or dynamic.

You turn any element into a flex container with one line:

.container {
  display: flex;
}

That's it. Every direct child of that element instantly becomes a flex item and lines up in a row by default.

Key idea: Flexbox works on one axis at a time — either a row (horizontal) or a column (vertical). For two-dimensional layouts, use CSS Grid instead.

2. Terminology

Understanding the two axes is crucial — all flexbox properties refer to them.

/* Main axis: the direction items are placed (row → horizontal, column → vertical) */
/* Cross axis: perpendicular to the main axis                                    */

.container {
  display: flex;             /* → main axis is horizontal by default */
  flex-direction: row;    /* main axis = left → right             */
}

/* flex-direction: column → main axis becomes top → bottom */
Flex Container

The parent element with display: flex. Controls how children are arranged.

Flex Items

The direct children of a flex container. Automatically become flexible.

Main Axis

The primary direction of layout. Set by flex-direction.

Cross Axis

Perpendicular to the main axis. If main axis is row → cross axis is column.

  flex-direction: column;        /* top to bottom               */
  flex-direction: column-reverse;/* bottom to top               */

Aligns items along the main axis. Controls how leftover space is distributed.

flex-wrap

By default, flex items all try to fit on one line. flex-wrap: wrap lets them break onto multiple lines.

.container {
  flex-wrap: nowrap;        /* all on one line (default)  */
  flex-wrap: wrap;          /* wrap to next line          */
  flex-wrap: wrap-reverse;  /* wrap in reverse order      */
}

gap (row-gap / column-gap)

Sets consistent space between flex items — much cleaner than using margins.

.container {
  gap: 1rem;           /* same gap for rows and columns  */
  gap: 1rem 2rem;     /* row-gap | column-gap           */
  row-gap: 1rem;      /* only vertical gaps             */
  column-gap: 2rem;   /* only horizontal gaps           */
}

align-content

Like justify-content but for rows when flex-wrap is on and there are multiple lines. Has no effect on single-line flex containers.

.container {
  align-content: flex-start;    /* rows packed at top         */
  align-content: center;        /* rows centered vertically   */
  align-content: flex-end;      /* rows packed at bottom      */
  align-content: space-between; /* space between rows         */
  align-content: space-around;  /* space around rows          */
  align-content: stretch;       /* rows stretch to fill       */
}

flex-flow (shorthand)

Combines flex-direction and flex-wrap into one property.

.container {
  /* flex-flow: <direction> <wrap> */
  flex-flow: row wrap;         /* most common combo            */
  flex-flow: column nowrap;     /* column, no wrapping          */
}

4. Item Properties Goes on the children

flex-grow

Controls how much an item grows to fill available space. Default is 0 (don't grow). A value of 1 means "take up all remaining space". If multiple items have flex-grow: 1, they share the space equally.

.item {
  flex-grow: 0;   /* don't grow (default)              */
  flex-grow: 1;   /* grow to fill remaining space       */
  flex-grow: 2;   /* grow twice as much as flex-grow: 1 */
}
Real-world use: Give a main content area flex-grow: 1 next to a fixed-width sidebar. The content fills all remaining space automatically.

flex-shrink

Controls how much an item shrinks when there isn't enough space. Default is 1 (can shrink). Set to 0 to prevent an item from shrinking.

.item {
  flex-shrink: 1;   /* shrinks if needed (default)     */
  flex-shrink: 0;   /* never shrinks — keeps its size  */
}

flex-basis

Sets the initial size of an item before remaining space is distributed. Think of it as the "starting width" (or height in a column). Overrides width on the main axis.

.item {
  flex-basis: auto;    /* uses the item's width/content (default) */
  flex-basis: 200px;  /* start at 200px before growing/shrinking */
  flex-basis: 30%;    /* start at 30% of the container           */
  flex-basis: 0;      /* start at nothing — distribute from zero */
}

flex (shorthand) ⭐ Most used

Combines flex-grow, flex-shrink, and flex-basis. Use this instead of setting them individually.

.item {
  /* flex: grow  shrink  basis */
  flex: 1 1 auto;    /* can grow, can shrink, auto size      */
  flex: 0 0 200px;  /* fixed 200px — no grow, no shrink    */
  flex: 1;           /* shorthand for flex: 1 1 0           */
  flex: auto;        /* shorthand for flex: 1 1 auto        */
  flex: none;        /* shorthand for flex: 0 0 auto        */
}
Common gotcha: flex: 1 sets flex-basis: 0, meaning items start from zero size. flex: auto sets flex-basis: auto, so items start from their content size. They look similar but behave differently.

align-self

Overrides align-items for a single item. Lets you position one item differently from the rest.

.special-item {
  align-self: auto;        /* follow parent's align-items (default) */
  align-self: flex-start;  /* align to start of cross axis          */
  align-self: center;      /* center on cross axis                  */
  align-self: flex-end;    /* align to end of cross axis            */
  align-self: stretch;     /* stretch to fill cross axis            */
}

order

Changes the visual order of items without changing the HTML. Default is 0. Lower numbers appear first.

.item-that-should-go-first {
  order: -1;   /* moves before all order: 0 items */
}

.item-that-should-go-last {
  order: 1;    /* moves after all order: 0 items  */
}
Accessibility note: order only changes visual order — screen readers still follow the HTML source order. Use it carefully.

5. Common Patterns

Perfect centering (horizontal + vertical)

The most famous flexbox trick — center anything inside a container with just 3 lines.

.center-me {
  display: flex;
  justify-content: center;   /* center on main axis (horizontal)  */
  align-items: center;       /* center on cross axis (vertical)   */
  height: 200px;             /* needs a height to center in!      */
}
I am perfectly centered!

Navigation bar (logo left, links right)

.nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}

.nav__links {
  display: flex;     /* nested flex for the links row */
  gap: 1.5rem;

Responsive card grid (wraps automatically)

.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1.5rem;
}

.card {
  flex: 1 1 280px;  /* grow and shrink, minimum 280px wide */
  /* Cards fill a row and wrap to next line when below 280px */
}

Button group

.btn-group {
  display: flex;
  gap: 1rem;
  flex-wrap: wrap;         /* stack buttons on small screens     */
  justify-content: center;  /* center on all screen sizes         */
}

Vertical stack (column layout)

.stack {
  display: flex;
  flex-direction: column;
  gap: 1rem;            /* consistent spacing between items */
}

/* Stick footer at bottom (requires a height on the parent) */
.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.page__footer {
  margin-top: auto;    /* pushes footer to the bottom       */
}

6. Quick Reference

Container Properties (parent)

Property Values What it does
display flex | inline-flex Activates flexbox on the container
flex-direction row | column | row-reverse | column-reverse Sets the main axis direction
justify-content flex-start | center | flex-end | space-between | space-around | space-evenly Aligns items along the main axis
align-items stretch | flex-start | center | flex-end | baseline Aligns items along the cross axis
flex-wrap nowrap | wrap | wrap-reverse Controls whether items wrap to new lines
gap any length value (e.g. 1rem) Space between flex items
align-content same as justify-content Aligns rows when wrapped (multi-line)
flex-flow <direction> <wrap> Shorthand for flex-direction + flex-wrap

Item Properties (children)

Property Values What it does
flex-grow number (default 0) How much the item grows to fill space
flex-shrink number (default 1) How much the item shrinks when short on space
flex-basis auto | length | 0 Starting size before grow/shrink apply
flex <grow> <shrink> <basis> Shorthand for all three above ⭐
align-self auto | flex-start | center | flex-end | stretch Override align-items for this item only
order integer (default 0) Change visual order without changing HTML

7. Flexbox vs CSS Grid

Both are powerful but solve different problems. The quick mental model:

Use Flexbox when...

You are arranging items in one direction — either a row or a column. Great for navbars, button groups, centering content, card rows, and stacking items vertically.

Use Grid when...

You need to control both rows and columns at the same time. Great for page layouts, image galleries, and any layout where you need precise two-dimensional placement.

/* Flexbox — one axis, content-driven sizing */
.flex-row {
  display: flex;
  gap: 1rem;
}

/* Grid — two axes, explicit track sizing */
.grid-layout {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  gap: 1rem;
}
They work great together: Use Grid for the big page layout (header, sidebar, content, footer) and Flexbox for the small components within each section (nav links, button groups, card content).
Rule of thumb: "If I didn't have Flexbox, I'd need floats or inline-block → use Flexbox. If I need a table-like layout of rows and columns → use Grid."
